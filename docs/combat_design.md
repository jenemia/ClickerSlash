# 전투 설계 문서

## 4레인 전투 구조

### 기본 배치
| 요소 | 설계 |
| --- | --- |
| 화면 방향 | 가로형 PC/Web |
| 공간 구성 | 3D 월드 전장, 고정 카메라 시점 |
| 플레이어 위치 | 화면 하단 고정 영역 |
| 적 위치 | 화면 상단 스폰 후 하단 방향으로 이동 |
| 레인 수 | 4개 고정 레인 |
| 승리 조건 | 60초 동안 생존 |
| 실패 조건 | 플레이어 라이프 0 |

### 레인 역할
| 레인 | 기능 |
| --- | --- |
| 1~4번 레인 | 적의 주 이동 경로이자 플레이어 대응 위치 |
| 하단 방어선 | 적이 이 선을 통과하면 라이프가 감소하는 기준선 |
| 하단 안전 구역 | 플레이어의 기본 대기 위치이자 자동공격 시작 지점 |

## 3D 전장 표현

### 카메라 기준
| 항목 | 설계 |
| --- | --- |
| 카메라 타입 | 고정 원근 카메라 |
| 시점 | 플레이어 뒤쪽 상단에서 레인을 내려다보는 각도 |
| 목표 | 상단 스폰 지점, 플레이어 위치, 하단 방어선을 한 화면에 유지 |
| 카메라 움직임 | 프로토타입에서는 없음 |
| 줌/연출 | 위험 연출이 있어도 플레이 공간 프레이밍은 유지 |

### 레인 시각화 기준
| 항목 | 설계 |
| --- | --- |
| 레인 바닥 | 3D 바닥 메쉬 위에 발광 스트립과 경계선 사용 |
| 방어선 | 하단에 명확한 발광 라인 또는 게이트 메쉬 배치 |
| 플레이어 높이 | 바닥과 분리된 낮은 높이로 고정해 실루엣 유지 |
| 적 가독성 | 체력/속도 타입이 색상, 실루엣, 크기로 구분되게 설계 |

### 3D 표현 원칙
- 전투의 핵심은 깊이감 자체가 아니라 레인 가독성이다.
- 배경 오브젝트는 월드 분위기를 만들되 레인 경계를 가리지 않아야 한다.
- 이펙트는 타격 지점, 하단 돌파, 위협 레인 강조에만 집중하고 화면 전체를 덮지 않는다.
- 적 크기 차이는 정보 전달용으로만 쓰고, 히트박스 오해를 만들 정도로 과장하지 않는다.

### 전투의 한 사이클
1. 적이 4개 레인 중 하나에 일정 간격으로 랜덤 스폰된다.
2. 적은 체력과 이동 속도에 따라 하단 방어선 방향으로 내려온다.
3. 플레이어는 좌우 버튼 입력으로 목표 레인에 이동한다.
4. 이동이 완료되면 플레이어는 현재 레인의 가장 앞 적을 자동으로 공격한다.
5. 적을 처치하면 콤보가 유지되고, 적이 하단 방어선을 통과하면 라이프가 1 감소하며 콤보가 종료된다.

## 플레이어 이동 방식

### 입력 규칙
| 항목 | 규칙 |
| --- | --- |
| 입력 방식 | 좌우 버튼으로 한 칸씩 이동 |
| 예약 입력 | 제한 없음 |
| 순간이동 | 없음 |
| 기본 이동 속도 | 고정값 |
| 성장 반영 | 패시브 업그레이드로 이동 속도 증가 가능 |
| 이동 중 공격 | 불가 |

### 추천 이유
- 좌우 이동만 이해하면 되므로 PC/Web에서도 조작 규칙이 단순하다.
- 입력 예약 제한이 없어서 급한 상황에서도 여러 칸 이동 의도를 미리 넣을 수 있다.
- 이동 중 공격이 끊기므로 레인 선택의 무게가 생기고, 자리 선점 판단이 전투의 핵심이 된다.
- ECS 관점에서는 `InputBuffer -> MoveQueue -> LaneMoveResolve` 흐름이 단순하고, 향후 이동 속도 증가나 이동 보조 패시브를 붙이기 쉽다.
- 3D 환경에서도 좌우 한 칸 이동 규칙이 유지되므로, 화면상 깊이감과 실제 입력 규칙이 충돌하지 않는다.

### 프로토타입 검증 포인트
| 항목 | 확인 내용 |
| --- | --- |
| 한 칸 이동 체감 | 4레인 구조에서 답답하지 않은지 |
| 예약 입력 체감 | 의도한 이동 경로가 자연스럽게 이어지는지 |
| 이동 중 공격 중단 | 플레이어가 레인 선점 리스크를 명확히 느끼는지 |

## 공격 시스템

### 기본 원칙
- 플레이어는 공격 버튼을 따로 누르지 않는다.
- 이동이 끝나면 즉시 자동공격이 시작된다.
- 자동공격은 현재 플레이어가 위치한 레인의 가장 앞 적을 우선 타격한다.
- 기본 공격 속도는 고정값이며, 후속 패시브 업그레이드로 변경 가능하다.

### 프로토타입 공격 규칙
| 항목 | 규칙 |
| --- | --- |
| 공격 시작 조건 | 레인 이동 완료 |
| 공격 대상 | 현재 레인의 가장 앞 적 |
| 공격 주기 | 기본 고정 속도 |
| 공격 중 레인 변경 | 이동 시작 시 공격 중단 |
| 타겟 재선정 | 현재 대상 처치 또는 레인 변경 시 재탐색 |

### 향후 확장 단서
- 전투 중 플레이어가 점점 강해질 수 있는 구조는 열어 두되, 프로토타입에서는 구체 수치와 발동 조건을 확정하지 않는다.
- 추후 패시브나 강화로 `관통 수치`, `인접 레인 공격`, `멀티레인 공격`이 가능해질 수 있게 설계한다.
- 현재 문서에서는 기본 전투 규칙만 확정하며, 확장 규칙은 ECS 구조와 메타 설계에서만 훅을 남긴다.

## 콤보 시스템

### 목적
- 적을 끊기지 않고 정리하는 운영 욕구를 만든다.
- 하단 돌파를 막는 플레이에 즉시적인 긴장감을 추가한다.
- 향후 전투 중 강화 시스템의 입력값이 될 수 있는 확장 포인트를 확보한다.

### 기본 규칙
| 항목 | 규칙 |
| --- | --- |
| 콤보 증가 | 적 처치 시 +1 |
| 콤보 유지 | 하단 돌파가 없으면 유지 |
| 콤보 종료 | 적이 하단 방어선을 통과할 때 초기화 |
| UI 표시 | 프로토타입에서는 수치만 표시 |

### 프로토타입 범위
- 현재 단계에서는 콤보 수치에 따른 추가 효과를 확정하지 않는다.
- 다만 문서와 ECS 구조에는 “콤보가 전투 중 강화 트리거가 될 수 있음”이라는 단서만 남긴다.

## 적 종류 설계
| 적 타입 | 특징 | 플레이어 요구 | ECS 데이터 포인트 |
| --- | --- | --- | --- |
| 저체력 고속형 | 빨리 내려오지만 쉽게 처치됨 | 빠른 레인 이동과 우선 대응 | `EnemyHealth`, `MoveSpeed` |
| 중간형 | 평균 체력, 평균 속도 | 기본 운영 확인용 | `EnemyHealth`, `MoveSpeed` |
| 고체력 저속형 | 천천히 오지만 오래 막힘 | 먼저 자리 잡고 공격 유지 필요 | `EnemyHealth`, `MoveSpeed` |

### 프로토타입 적 설계 원칙
- 프로토타입에서는 적이 공격하지 않는다.
- 적 종류 차이는 `체력`과 `이동 속도`만으로 만든다.
- 별도 상태이상, 특수 패턴, 레인 변경, 버프/디버프 적은 넣지 않는다.
- 3D 모델 차이는 정보 전달 수준으로만 주고, 공격 판정이나 충돌 규칙은 동일한 레인 기준으로 유지한다.

## 스테이지 구조

### 프로토타입 스테이지 템플릿
| 구간 | 길이 | 목적 |
| --- | --- | --- |
| 도입 구간 | 0~15초 | 좌우 이동과 자동공격 기본 감각 학습 |
| 압박 구간 | 15~40초 | 레인 전환과 우선순위 판단 압박 |
| 밀도 상승 구간 | 40~60초 | 라이프와 콤보 유지 압박 확인 |

### 종료 조건
- 한 판은 60초 생존전으로 고정한다.
- 시간이 끝날 때 라이프가 1 이상이면 승리다.
- 라이프가 0이 되면 즉시 실패한다.

## 전투 템포
| 요소 | 기준 |
| --- | --- |
| 전투 길이 | 60초 |
| 스폰 간격 | 일정 간격의 데이터 조정 가능 값 |
| 플레이어 이동 속도 | 고정값 |
| 플레이어 공격 속도 | 고정값 |
| 난이도 상승 방식 | 스폰 밀도와 적 체력/속도 조합 증가 |

### 템포 설계 원칙
- 전투는 빠르되, 플레이어가 “어느 레인으로 갈지” 판단할 시간은 남겨야 한다.
- 초반에는 한 번에 여러 레인을 동시에 비우게 만드는 극단 스폰을 피한다.
- 중후반에는 저체력 고속형과 고체력 저속형을 섞어 우선순위 판단을 유도한다.
- 3D 카메라 원근으로 속도감이 과장되어 보일 수 있으므로, 실제 이동 속도와 화면 체감 속도를 함께 튜닝한다.

## 리듬감 강화 요소
| 요소 | 목적 |
| --- | --- |
| 일정한 스폰 간격 | 자동공격 전투에도 반복 템포를 느끼게 함 |
| 레인 점멸 | 활성 위협 레인을 빠르게 읽게 함 |
| 타격 사운드 레이어 | 연속 처치와 콤보 유지 쾌감 강화 |
| 하단 돌파 경고 연출 | 라이프 손실 직전의 긴장감 강조 |
| 콤보 UI 강조 | 끊기지 않는 운영 욕구 강화 |
| 3D 깊이 라이트 | 상단에서 내려오는 적의 거리감을 자연스럽게 읽게 함 |

## 초반 1개 스테이지 예시

### Stage 1: “Line Hold”
| 시간 | 내용 | 학습 포인트 |
| --- | --- | --- |
| 0~10초 | 중간형이 4개 레인에 낮은 밀도로 랜덤 등장 | 좌우 이동과 자동공격 이해 |
| 10~25초 | 저체력 고속형 비중 증가 | 빠른 레인 전환과 예약 입력 체감 |
| 25~40초 | 고체력 저속형과 저체력 고속형 혼합 | 어느 레인을 먼저 지킬지 우선순위 판단 |
| 40~55초 | 스폰 밀도 상승 | 이동 중 공격 중단 리스크 체감 |
| 55~60초 | 마지막 압박 구간 | 라이프와 콤보 동시 방어 |

### Stage 1 보상 의도
- 첫 완주 보상으로 메타 트리의 첫 분기 1개를 열게 한다.
- 플레이어가 “더 강해졌다”보다 “다음에는 더 안정적으로 레인을 운영할 수 있다”는 기대를 갖게 해야 한다.

## 실행 우선순위
1. 좌우 이동, 무제한 입력 예약, 이동 중 공격 불가 규칙부터 구현한다.
2. 이동 완료 후 자동공격과 현재 레인의 가장 앞 적 타겟팅을 먼저 검증한다.
3. 저체력 고속형, 중간형, 고체력 저속형 3종 적으로 첫 스테이지를 구성한다.
4. 60초 생존전, 3 라이프, 하단 돌파 시 콤보 종료 규칙을 함께 검증한다.
5. 고정 3D 카메라와 레인 가시성이 전투 판단을 방해하지 않는지 먼저 확인한다.
