# 전체 인게임 구조 문서

## 목적
- 이 문서는 현재 프로토타입의 인게임 전체 구조를 레인, 적재장, 허브 연계까지 한 번에 설명한다.
- 기준 범위는 `PrototypeHub` -> `PrototypeBattle` -> 결과/복귀 루프다.

## 큰 구조
1. 허브에서 스킬트리와 현재 상태를 확인한다.
2. 전투 씬으로 들어가 레인 구역에서 물류를 처리한다.
3. 성공 처리된 물류는 상하차 큐에 동시에 누적된다.
4. 필요할 때 `Q`로 적재장으로 이동해 누적 물류를 클릭 처리한다.
5. `Esc`로 일시정지 팝업을 열고 재개하거나 허브로 돌아갈 수 있다.
6. 세션 종료 후 결과를 허브가 읽는다.

## 씬 구조
- `PrototypeHub`
  - 메타 진행과 스킬트리 UI를 보여준다.
- `PrototypeBattle`
  - 레인 전투, 적재장, HUD, 카메라 전환, 미니게임 프레젠터를 가진다.

## 전투 세션 구조
- 레인 구역
  - 기본 작업 공간
  - 플레이어 직접 이동과 물류 처리
- 적재장 구역
  - 레인 성공 처리 결과를 따로 모아 처리하는 후속 작업 공간
  - 고정 5슬롯 + backlog 구조
- 전역 런타임
  - `PrototypeSessionRuntime`가 메타 상태, 세션 큐, 상하차 전환 상태, 결과 스냅샷을 유지한다.

## 작업 구역 전환
- 기본 상태는 `Lane`
- `Q`
  - 레인에서는 적재장 진입 요청
  - 적재장에서는 레인 복귀 요청
- 전환 단계
  - `EnteringLoadingDock`
  - `ActiveInLoadingDock`
  - `ReturningToLane`
- 카메라
  - `LaneVirtualCamera`와 `LoadingDockVirtualCamera` 두 개를 `CinemachineBrain` 블렌드로 전환한다.
  - 카메라 위치 값은 씬에 배치된 virtual camera 자체가 기준이다.

## 입력 구조
- 레인 이동
  - `A / D`, `Left / Right`
- 적재장 전환
  - `Q`
- 일시정지
  - `Esc`
- 적재장 처리
  - 마우스 좌클릭

## 상태 잠금 규칙
- 상하차 진입, 체류, 복귀 중에는 레인 플레이어 이동이 잠긴다.
- 잠금 의미는 다음 세 가지를 모두 포함한다.
  - 새 이동 입력 차단
  - 대기 중 이동 큐 삭제
  - 진행 중 보간 이동 취소 후 현재 확정 레인으로 스냅
- 일시정지 팝업이 열려 있으면 적재장 입력과 구역 토글도 막힌다.

## 레인과 적재장의 연결
- 레인에서 물류 성공 처리
  - 돈/콤보 증가
  - `CargoHandledEvent`
  - 같은 타입이 적재장 큐에 적재
- 레인에서 물류 실패 처리
  - 패널티 적용
  - 콤보 초기화
  - 적재장 큐에는 넣지 않음
- 적재장 뷰
  - 현재 구역과 상관없이 세션 큐를 항상 반영
  - 입력만 적재장 active 상태에서 허용

## HUD와 팝업
- 레인 기본 HUD
  - 남은 시간, 돈, 콤보, 레인 정보
- 적재장 HUD
  - active slot 수, backlog 수, 현재 적재 물류 목록
- 일시정지 팝업
  - `재개하기`
  - `허브로가기`

## 현재 구현 기준 요약
- 레인 물류 타입과 적재장 물류 타입은 공유 프리팹으로 통일되어 있다.
- 적재장은 고정 슬롯 구조이며 cargo가 자동 재정렬되지 않는다.
- 레인 상태에서도 적재장 3D 뷰는 큐 상태를 유지한다.
- 적재장 상세 상호작용은 현재 클릭 delivered만 구현되어 있다.
