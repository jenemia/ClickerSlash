# 스킬 트리 컨셉 문서

## 문서 목적
이 문서는 물류센터 게임의 메타 성장 축을 **체력 / 근력 / 이동 / 숙련 / 경영 / 자동화**의 6개 큰 줄기로 정리한 컨셉 문서다.

## 스킬 트리 공통 원칙
- 단순 수치 상승만이 아니라 **플레이 방식이 바뀌는 노드**를 포함한다.
- 초반에는 플레이어 능력 강화 위주, 중후반에는 센터 운영과 자동화 확장 위주로 전환한다.
- 레인 수 확장, 정보 제공, 자동화는 플레이 피로도를 줄이는 해금으로 묶는다.
- 특정 줄기만 과하게 올려도 플레이는 가능하지만, 최적 효율은 여러 줄기 조합에서 나오게 한다.

## 전체 트리 요약
| 줄기 | 역할 | 대표 판타지 |
| --- | --- | --- |
| 체력 | 한 판의 물류 작업시간 확장 | 더 오래 버티며 더 많은 물류를 소화한다 |
| 근력 | 더 무거운 물류 처리 | 힘으로 센터의 처리 한계를 넓힌다 |
| 이동 | 더 빠르게 레인을 오간다 | 순간 대응과 라인 커버 능력을 만든다 |
| 숙련 | 판정과 콤보를 안정화 | 같은 손놀림으로 더 높은 수익을 낸다 |
| 경영 | 수익 구조와 센터 확장 | 더 많은 라인과 더 나은 정보로 운영한다 |
| 자동화 | 설비가 실수를 보완 | 사람의 한계를 기계로 메운다 |

---

## 1. 체력 줄기
### 줄기 목적
- 스킬 트리에서 체력을 올려 한 세션의 물류 작업 가능 시간을 늘린다.
- 초반에는 짧고 빠른 세션, 후반에는 더 오래 운영하는 센터장 감각을 만든다.

### 핵심 효과
- `SessionDuration` 증가
- 세션 후반 진입 기회 확대
- 장시간 운영에 맞춘 고가 화물 노출 기회 증가

### 대표 노드
| 노드명 | 효과 | 분류 |
| --- | --- | --- |
| 기초 체력 단련 | 세션 작업시간 +5초 | 수치형 |
| 교대 적응 훈련 | 세션 작업시간 추가 증가 | 수치형 |
| 호흡 조절 | 피크타임 진입 전 집중 페널티 완화 | 보정형 |
| 연장 근무 승인 | 일정 조건에서 세션 종료 직전 작업시간 소폭 연장 | 규칙형 |
| 장기 운영 체질 | 긴 세션 구간에서 고가 화물 출현 보정 | 규칙형 |
| 현장 강행군 | 일정 콤보 이상 유지 시 소량의 작업시간 보너스 획득 | 해금형 |

### 플레이 스타일
- 더 긴 작업시간 동안 누적 수익을 크게 가져가는 지구력형 운영.
- 근력과 숙련이 받쳐주면 긴 세션 후반의 고수익 구간을 가장 잘 활용한다.

### 설계 주의점
- 체력은 전투식 생존력이 아니라 **물류 작업 지속 시간**으로만 해석되어야 한다.
- 체력만 올려도 무조건 강해지지 않도록 후반 구간 난도 상승과 다른 줄기 의존성을 유지한다.

---

## 2. 근력 줄기
### 줄기 목적
- 플레이어가 직접 처리 가능한 최대 무게를 높인다.
- 고중량 화물의 보상을 크게 먹는 빌드를 만든다.

### 핵심 효과
- `MaxHandleWeight` 증가
- 고중량 화물 성공 보상 배율 증가
- 한시적 과중량 처리 같은 특수 규칙 해금

### 대표 노드
| 노드명 | 효과 | 분류 |
| --- | --- | --- |
| 기초 근력 훈련 | 최대 처리 가능 무게 +2 | 수치형 |
| 작업 자세 교정 | 처리 중 경직 시간 소폭 감소 | 수치형 |
| 고중량 숙련 | Weight 12 이상 화물 보상 +10% | 규칙형 |
| 파워 리프트 | 세션당 1회 무게 초과 화물 처리 가능 | 액티브형 |
| 강철 허리띠 | 고중량 화물 Miss 페널티 일부 감소 | 보정형 |
| 특수 하역 면허 | 초대형 화물 등장 해금 | 해금형 |

### 플레이 스타일
- 무거운 물류를 중심으로 큰 돈을 노리는 고위험 고보상 운영.
- 이동이 부족하면 큰 기회를 놓치기 쉽다.

### 설계 주의점
- 근력만 올려도 모든 상황이 쉬워지면 안 된다.
- 무게 처리 외에도 이동/숙련이 필요하도록 스폰 패턴을 유지한다.

---

## 3. 이동 줄기
### 줄기 목적
- 더 빠르게 레인을 이동해 더 많은 물류를 커버한다.
- 3레인 이상부터 증가하는 피지컬 부담을 완화한다.

### 핵심 효과
- `LaneMoveDuration` 감소
- 연속 이동 가속
- 긴급 대응용 순간 이동류 해금

### 대표 노드
| 노드명 | 효과 | 분류 |
| --- | --- | --- |
| 발놀림 강화 | 인접 레인 이동 시간 감소 | 수치형 |
| 퀵 스텝 | 연속 이동 시 두 번째 이동 추가 가속 | 규칙형 |
| 출발 반응 향상 | 입력 지연 보정 | 보정형 |
| 긴급 대시 | 긴 쿨다운으로 임의 레인 즉시 이동 | 액티브형 |
| 작업 동선 최적화 | 먼 레인 이동 패널티 완화 | 규칙형 |
| 경량 장비화 | 처리 후 다음 이동 시작 속도 증가 | 수치형 |

### 플레이 스타일
- 많은 화물을 빠르게 먹으며 콤보를 유지하는 스피드형 운영.
- 무거운 화물 특화보다는 넓은 커버 범위에 강하다.

### 설계 주의점
- 이동이 지나치게 빨라지면 레인 판단이 무의미해진다.
- “잘 선택해서 움직이는 재미”가 남아 있도록 상한을 둔다.

---

## 4. 숙련 줄기
### 줄기 목적
- 판정 안정성과 콤보 유지력을 높인다.
- 같은 물류를 처리해도 더 높은 효율을 내는 숙련형 빌드를 만든다.

### 핵심 효과
- `TimingWindow` 확장
- 콤보 배율 상승
- Miss 완화 및 실수 보정

### 대표 노드
| 노드명 | 효과 | 분류 |
| --- | --- | --- |
| 작업 감각 향상 | Perfect 판정 범위 증가 | 수치형 |
| 숙련자 리듬 | 콤보 배율 상승 속도 증가 | 수치형 |
| 안전 처리 | Miss 시 손실 감소 | 보정형 |
| 재정렬 반사신경 | 세션당 1회 Late를 Good으로 보정 | 규칙형 |
| 집중 모드 | 특정 콤보 이상에서 판정 완화 | 해금형 |
| 클레임 최소화 | 고가 화물 Miss 패널티 감소 | 규칙형 |

### 플레이 스타일
- 안정적이고 균형 잡힌 수익을 내는 정교형 운영.
- 손실이 적어 장기적으로 매우 효율적이다.

### 설계 주의점
- 숙련 줄기가 지나치게 강하면 근력/이동 선택 의미가 약해진다.
- 완전 무실수 보장이 아니라 “위험 완화” 수준이어야 한다.

---

## 5. 경영 줄기
### 줄기 목적
- 돈을 더 잘 벌고, 더 많은 라인을 열고, 더 많은 정보를 받아 운영 자체를 바꾼다.
- 플레이어 능력 강화가 아니라 **센터 규모와 운영 도구 확장**을 담당한다.

### 핵심 효과
- 수익 배율 증가
- 레인 3/4/5 해금
- 사전 정보 제공
- 특수 계약 해금

### 대표 노드
| 노드명 | 효과 | 분류 |
| --- | --- | --- |
| 성과급 계약 | 모든 성공 처리 보상 +5% | 수치형 |
| 라인 증설 I | 3번째 레인 해금 | 해금형 |
| 라인 증설 II | 4번째 레인 해금 | 해금형 |
| 라인 증설 III | 5번째 레인 해금 | 해금형 |
| 사전 표시 모니터 | 다음 화물 1개 미리 표시 | 규칙형 |
| 우선순위 표시기 | 고가/고위험 화물 강조 표시 | 규칙형 |
| 피크타임 계약 | 세션 후반 20% 구간 고가 화물 출현율 증가 | 해금형 |

### 플레이 스타일
- 수익 최적화와 운영 정보 활용에 강한 센터장형 빌드.
- 레인이 늘수록 이동/숙련과의 시너지가 크다.

### 설계 주의점
- 레인 해금은 반드시 부담 완화 도구와 같이 배치한다.
- 정보 제공이 너무 많아 화면이 복잡해지지 않도록 단계적으로 연다.

---

## 6. 자동화 줄기
### 줄기 목적
- 사람이 놓친 일부 물류를 기계가 보완하게 한다.
- 후반으로 갈수록 “내가 직접 다 하지 않아도 되는 센터” 감각을 만든다.

### 핵심 효과
- Miss 보완
- 특정 화물 자동 처리
- 무게/가치 자동 스캔
- 레인 전용 설비 해금

### 대표 노드
| 노드명 | 효과 | 분류 |
| --- | --- | --- |
| 자동 스캐너 | 모든 화물 무게 표시 | 정보형 |
| 리턴 벨트 | Miss 화물을 낮은 확률로 재투입 | 규칙형 |
| 보조 암 설치 | 저효율 자동 처리 시도 | 규칙형 |
| 하중 분산 장치 | 고중량 화물 처리 시간 감소 | 수치형 |
| 레인 보조 로더 | 지정 레인의 저중량 화물 자동 처리 | 해금형 |
| 우선 처리 드론 | 세션당 1회 가장 위험한 화물 자동 회수 | 액티브형 |

### 플레이 스타일
- 피지컬 부담을 줄이고 안정 수익을 만드는 설비형 운영.
- 후반 다레인 환경에서 강력한 유지력을 가진다.

### 설계 주의점
- 자동화가 플레이어를 완전히 대신하면 게임이 방치형으로만 변한다.
- 초반 자동화는 “실수 보완”, 후반 자동화는 “부분 자동 처리” 순으로 단계 상승시킨다.

---

## 추천 해금 흐름
### 초반
- 체력 1단계
- 근력 1단계
- 이동 1단계
- 숙련 1단계
- 경영의 수익 노드

### 중반
- 체력 2단계
- 경영의 3레인 해금
- 숙련의 콤보 강화
- 이동의 긴급 대시
- 자동화의 스캐너

### 후반
- 경영의 4~5레인 해금
- 자동화의 리턴 벨트/보조 암
- 근력의 특수 하역 면허
- 경영의 특수 계약류

## 줄기 간 추천 시너지
| 조합 | 성향 |
| --- | --- |
| 체력 + 숙련 | 긴 세션에서 안정적으로 수익을 누적하는 운영형 |
| 체력 + 경영 | 긴 작업시간과 후반 보상 구간을 함께 활용하는 센터 확장형 |
| 근력 + 숙련 | 무거운 화물을 안정적으로 먹는 고수익형 |
| 이동 + 숙련 | 콤보 중심의 손맛형 |
| 경영 + 자동화 | 넓은 레인과 운영 정보로 굴리는 센터장형 |
| 근력 + 이동 | 고중량도 놓치지 않으려는 공격적 운영형 |

## 최종 설계 원칙
1. 초반에는 플레이어 신체 능력 강화가 체감되어야 한다.
2. 체력은 전투형 HP가 아니라 세션 작업시간 증가 축으로 명확히 구분되어야 한다.
3. 중반부터는 레인 수 증가와 정보 제공으로 게임 구조가 바뀌어야 한다.
4. 후반에는 자동화가 실수를 줄여주되 플레이 자체를 지우면 안 된다.
5. 최고의 빌드는 한 줄기 몰빵이 아니라 여러 줄기 조합에서 나오게 설계한다.
