# 게임 시스템 구현용 인게임 기획서

## 1. 문서 목적
이 문서는 **물류센터 컨셉의 인게임 시스템을 Unity ECS(DOTS)로 구현할 수 있도록** 핵심 규칙, 상태 정의, 판정 구조, 데이터 기준, 시스템 분리를 정리한 문서다.

## 2. 게임 개요
| 항목 | 내용 |
| --- | --- |
| 장르 | 세로형 레인 리듬 운영 게임 |
| 세션 길이 | 플레이어 체력 데이터 기반 작업시간 |
| 시작 레인 수 | 2개 |
| 최대 레인 수 | 5개 |
| 플레이 목표 | 세션 종료 시 정산 금액 최대화 |
| 기본 조작 | 레인 선택 이동 |
| 메인 리소스 | 돈, 콤보, 최대 처리 가능 무게, 작업 가능 시간 |

## 3. 핵심 규칙
1. 상단 스폰 지점에서 물류가 생성되어 하단 처리 구역으로 이동한다.
2. 플레이어는 한 시점에 하나의 레인 위치만 점유한다.
3. 물류가 처리 구간에 도달했을 때 플레이어가 같은 레인에 있고, 현재 처리 가능 무게 이상이 아니며, 다른 물류를 처리 중이 아니면 처리 성공 판정이 발생한다.
4. 처리 성공 시 돈이 증가하고 콤보가 오른다.
5. 처리 실패 또는 미처리 통과 시 돈이 감소하고 콤보가 끊긴다.
6. 세션은 전투식 체력 소모 게임오버가 아니라 **현재 체력 데이터로 산정된 작업시간 종료 후 정산**으로 종료한다.

## 4. 플레이어 조작 설계
### 4.1 입력 방식
- 모바일: 하단 레인 버튼 터치 또는 레인 영역 직접 터치
- 키보드: `1~5` 직접 레인 선택, 보조로 `A/D` 또는 좌우키 순차 이동 지원
- 내부 로직은 모두 `TargetLaneIndex` 변경으로 통일한다.

### 4.2 이동 규칙
- 플레이어는 현재 레인에서 목표 레인으로 이동한다.
- 이동 시간은 `인접 레인 수 x 기본 이동 시간`으로 계산한다.
- 초기 프로토타입 기준 인접 레인 1칸 이동 시간은 **0.18초**를 사용한다.
- 이동 중에는 다른 물류를 처리할 수 없다.

### 4.3 처리 방식
- 별도 공격 버튼 없이 **처리 구간 도달 시 자동 처리**가 기본이다.
- 리듬감은 버튼 연타가 아니라 **정확한 도착 타이밍**에서 발생한다.
- 플레이어가 너무 빨리 도착해도 대기 가능하지만, 너무 늦게 도착하면 Miss가 난다.

## 5. 전장 구성
| 요소 | 설명 |
| --- | --- |
| Spawn Line | 물류가 생성되는 상단 라인 |
| Travel Zone | 물류가 이동하는 일반 구간 |
| Judgment Zone | 플레이어가 물류를 처리할 수 있는 하단 판정 구간 |
| Fail Line | 이 지점을 지나면 미처리 실패 |
| Lane Index | 각 컨베이어 라인의 정수 인덱스 |

### 5.1 레인 수 규칙
- 새 계정/새 세이브는 2레인으로 시작한다.
- 경영 스킬 트리에서 3, 4, 5레인을 순차 해금한다.
- 세션 시작 시 현재 해금된 레인 수만큼만 스폰 테이블이 활성화된다.

## 6. 물류 설계
### 6.1 물류 공통 속성
| 필드 | 설명 |
| --- | --- |
| LaneIndex | 어느 레인에서 내려오는지 |
| Weight | 물류 무게 |
| BaseReward | 성공 시 기본 보상 |
| BasePenalty | 실패 시 기본 손실 |
| MoveSpeed | 하강 속도 |
| SpawnTime | 생성 시간 |
| CargoType | 일반/긴급/고가/파손주의 등 |
| Flags | 특수 규칙 비트 플래그 |

### 6.2 MVP 물류 타입
| 타입 | 특징 | MVP 포함 여부 |
| --- | --- | --- |
| 일반 화물 | 기본 보상/손실만 존재 | 포함 |
| 고중량 화물 | 높은 무게, 높은 보상 | 포함 |
| 긴급 화물 | 빠른 속도, 짧은 판정 여유 | 선택 |
| 파손 주의 화물 | 실패 패널티 큼 | 추후 |
| 고가 화물 | 보상과 손실 모두 큼 | 추후 |

### 6.3 무게 기준
- 내부 계산은 정수 `Weight` 값을 사용한다.
- UI 표시는 kg 느낌으로 표현 가능하지만, 밸런스는 정수 단위로 잡는다.
- 초기 플레이어 기본 최대 처리 가능 무게는 **10**이다.
- 초기 MVP 화물 무게 범위는 **3 ~ 14**를 추천한다.

## 7. 판정 구조
### 7.1 처리 성공 조건
아래 조건을 모두 만족하면 성공 처리 후보가 된다.
1. 물류가 `Judgment Zone` 안에 있다.
2. 플레이어 `CurrentLaneIndex` == 물류 `LaneIndex`
3. 플레이어 `MaxHandleWeight >= CargoWeight`
4. 플레이어가 `Busy` 상태가 아니다.

### 7.2 판정 등급
| 판정 | 조건 예시 | 효과 |
| --- | --- | --- |
| Perfect | 물류 중심과 플레이어 도착 타이밍 차이가 매우 작음 | 보상 x1.2, 콤보 +1 |
| Good | 판정 구간 내 정상 처리 | 보상 x1.0, 콤보 +1 |
| Late | 처리 시작은 했지만 타이밍이 늦음 | 보상 x0.7, 콤보 유지 또는 약화 |
| Miss | 처리 실패, 무게 초과, 통과 | 손실 발생, 콤보 0 |

### 7.3 처리 시간
- 처리 시간은 무게에 따라 증가한다.
- 프로토타입 식: `HandleDuration = 0.18 + Weight * 0.015`
- 처리 시간 동안 플레이어는 다음 물류를 처리할 수 없다.
- 이 시간이 무거운 물류와 빠른 물류 사이의 운영 판단을 만든다.

## 8. 수익/손실 규칙
### 8.1 보상 계산식
`EarnedMoney = BaseReward x TimingMultiplier x ComboMultiplier x RewardModifier`

### 8.2 손실 계산식
`LostMoney = BasePenalty x PenaltyModifier`

### 8.3 초기 밸런스 예시
- `BaseReward = Weight x 10`
- `BasePenalty = Weight x 6`
- `Perfect TimingMultiplier = 1.2`
- `Good TimingMultiplier = 1.0`
- `Late TimingMultiplier = 0.7`
- `ComboMultiplier = 1.0 + min(Combo * 0.03, 0.6)`

### 8.4 콤보 규칙
- 성공 처리 시 콤보 +1
- Miss 시 콤보 0
- 콤보는 수익 보정의 핵심이다.
- 숙련 트리에서 Miss 시 콤보 일부 유지, 콤보 상승 속도 증가 등을 확장한다.

## 9. 세션 진행 구조
### 9.1 시간대 페이싱
| 구간 | 진행도 기준 | 목적 |
| --- | --- | --- |
| 워밍업 | 시작~25% | 2레인/가벼운 화물 중심으로 기본 리듬 인지 |
| 혼합 구간 | 25%~58% | 가벼운 화물과 고중량 화물 혼합 |
| 피크타임 | 58%~83% | 스폰 간격 감소, 우선순위 판단 강조 |
| 정산 직전 | 83%~100% | 높은 가치 화물 등장 가능, 마지막 판단 압박 |

### 9.2 세션 종료
- 현재 체력으로 산정된 작업시간이 끝나면 더 이상 신규 화물을 생성하지 않는다.
- 화면 안에 남아 있는 화물은 짧은 정리 페이즈 후 자동 정산한다.
- 최종 결과 화면에서 획득 돈, 최고 콤보, 처리 성공률, 놓친 화물 수를 보여준다.

## 11. 메타 성장과 인게임 연결
| 메타 요소 | 인게임 반영 |
| --- | --- |
| 체력 강화 | `SessionDuration` 또는 시작 작업시간 증가 |
| 근력 강화 | `MaxHandleWeight` 증가 |
| 이동 강화 | `LaneMoveDuration` 감소 |
| 숙련 강화 | 판정 범위 증가, Miss 보정 |
| 경영 강화 | 레인 해금, 수익 배율, 미리보기 정보 |
| 자동화 강화 | 놓친 화물 보조 처리, 자동 스캔 |

## 12. 자동화 설비 규칙
### 12.1 기본 철학
- 자동화는 플레이어를 대체하는 것이 아니라 **실수 보완**에서 시작한다.
- 초반 자동화는 `Miss`를 일부 완화하는 수준으로 제한한다.

### 12.2 예시 설비
| 설비 | 기능 |
| --- | --- |
| 리턴 벨트 | Miss 난 화물을 낮은 확률로 한 번 더 올려보냄 |
| 보조 암 | 지나간 화물을 낮은 효율로 대신 처리 |
| 자동 스캐너 | 무게/가치 정보를 일찍 표시 |
| 레인 보조 로더 | 지정 레인의 저중량 화물을 자동 처리 |

### 14.4 처리 판정 흐름
1. Cargo가 판정 구간에 진입한다.
2. 같은 레인에 있는 Worker를 찾는다.
3. Worker가 Busy가 아니고, 무게 조건을 만족하면 처리 시작 이벤트를 만든다.
4. 타이밍 차이에 따라 Perfect/Good/Late를 결정한다.
5. 보상과 콤보를 반영한다.
6. 처리 실패 또는 판정 구간 이탈 시 Miss 이벤트를 만든다.
7. Miss 이벤트는 손실/콤보 초기화/자동화 보완 순으로 처리한다.

## 15. 구현 시 주의점
- 이동과 판정은 프레임 의존이 아니라 **시간 기반 값**으로 계산해야 한다.
- 레인이 늘어날수록 입력 피로가 급증하므로 HUD 정보와 자동화 해금이 함께 붙어야 한다.
- 자동화가 너무 빨리 강해지면 플레이 의미가 사라진다.
- 무게 수치만 올리면 메타가 평평해지므로, 정보 제공과 규칙 변화형 해금을 조기에 넣어야 한다.

## 16. 1차 프로토타입 검증 질문
1. 2레인만으로도 레인 이동 타이밍이 재미있는가?
2. 자동 처리 방식만으로도 리듬감이 살아나는가?
3. 고중량 화물을 포기할지 먹을지 고민이 실제 판단을 만드는가?
4. 체력 기반 작업시간 종료 시 “한 판 더” 동기와 성장 체감이 생기는가?

## 17. 역할 승계형 튜토리얼/메타 내러티브
플레이어 성장과 물류센터 확장을 하나의 승계 구조로 묶는다.

### 17.1 핵심 컨셉
- 초반: **사수(멘토)**가 플레이어에게 분류 실무를 가르친다.
- 중반: 플레이어가 사수급이 되면 기존 사수는 **사장**으로 승격되어 센터 운영(증설/자동화)을 가르친다.
- 후반: 플레이어가 사장 업무까지 마스터하면 기존 사장은 **회장**이 되고, 플레이어는 새로운 **사장 역할**을 맡는다.

즉, 플레이는 반복이 아니라 **직급 상승에 따른 책임 확장**으로 느껴지게 만든다.

### 17.2 단계별 플레이 포커스
| 단계 | 플레이어 역할 | 멘토 역할 | 핵심 학습 | 해금 요소 |
| --- | --- | --- | --- | --- |
| T1 | 신입 분류원 | 사수 | 기본 입력, 타이밍 판정, 콤보 유지 | 2~3레인, 기본 화물 |
| T2 | 사수급 오퍼레이터 | 사장(승격된 사수) | 피크타임 운영, 우선순위 판단 | 4~5레인, 특수 화물 |
| T3 | 사장 | 회장(승격된 사장) | 레인 증설, 자동화 배치, 수익 최적화 | 센터 증축, 고급 자동화 |

### 17.3 튜토리얼 연출 구조
1. **사수 튜토리얼(조작)**
   - 입력 실수 허용 폭을 넓게 시작
   - 사수 콜아웃: “지금은 2번 레인”, “긴급 먼저 처리”
2. **사장 튜토리얼(운영)**
   - 웨이브 사이 운영 의사결정 제시
   - 예: 레인 증설 vs 자동분류기 업그레이드
3. **회장 단계(전략)**
   - 장기 KPI 중심 가이드
   - 예: 시간당 처리량, 누적 불량률, 자동화 효율

### 17.4 능력치/역할 연동 규칙
- 플레이어가 사수급으로 승급하면 기존 성장 능력치는 유지된 채 상위 단계 규칙이 열린다.
- 승급 시 추가 패시브를 부여한다.
  - 예: `판정 안정도 +X`, `초기 콤보 +Y`, `긴급 화물 가시성 +Z`
- 역할이 올라갈수록 **손속도 스킬 + 운영 스킬**의 비중을 함께 높인다.

### 17.5 클리커 메타와의 연결
- 사수 단계: 분류 정확도와 손속도 중심 성장
- 사장 단계: 레인 증설, 자동화, 설비 투자 중심 성장
- 회장 단계: 다센터 보너스, 전역 버프, 프레스티지 설계

### 17.6 UX 가이드
- HUD/튜토리얼 텍스트에서 현재 직급과 다음 승급 목표를 항상 노출한다.
- 승급 연출은 짧고 강하게(5~8초), 이후 즉시 새 시스템 체험으로 이어지게 한다.
- 내러티브 텍스트는 짧게 유지하고, 실제 학습은 플레이 상황 콜아웃으로 전달한다.
