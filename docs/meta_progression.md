# 메타 성장 설계 문서

## 전투 보상

### 기본 보상 구조
| 보상 | 획득 조건 | 용도 |
| --- | --- | --- |
| `Core Scrap` | 모든 전투 기본 보상 | 범용 노드 해금, 기초 성장 |
| `Pulse Data` | 생존 시간, 적 처치 수, 콤보 유지 성과 | 전투 운영 관련 노드 강화 |
| `Boss Sigil` | 보스 또는 엘리트 처치 | 상위 트리 해금, 희귀 노드 구매 |
| `Pattern Shard` | 특정 조건 달성형 전투 성과 | 대응 방식 변조형 노드 개방 |
| `Idle Charge` | 오프라인 누적 | 재화 보조, 자동화 보강 |

### 보상 배수 입력값
| 요소 | 보상에 미치는 영향 |
| --- | --- |
| 생존 시간 | 기본 지급량 결정 |
| 적 처치 수 | `Pulse Data` 증가 |
| 최대 콤보 | 보조 재화 및 희귀 조각 증가 |
| 남은 라이프 | 완주 효율 보정 |
| 메타 노드 효과 | 재화 종류별 획득량 변형 |

## 아웃게임 재화
| 재화 | 성격 | 역할 |
| --- | --- | --- |
| `Core Scrap` | 저가치 대량 재화 | 기초 성장, 초반 해금 |
| `Pulse Data` | 전투 운영 기반 재화 | 이동/공격/콤보 특화 노드 |
| `Boss Sigil` | 희귀 재화 | 상위 노드, 특성 슬롯, 고급 분기 |
| `Archive Key` | 조건 달성형 해금 재화 | 신규 적 프리셋, 신규 스테이지, 특수 규칙 개방 |

## 영구 성장 구조

### 구조 원칙
- 영구 성장은 공격력/체력 수치만 키우는 계단형이 아니라, **전투 규칙을 비틀 수 있는 노드**를 우선한다.
- 성장은 크게 세 갈래다.
| 갈래 | 설명 |
| --- | --- |
| 입력 완화형 | 이동 속도, 입력 보정, 레인 전환 안정성 |
| 전투 확장형 | 공격 속도, 공격 범위, 콤보 연계 가능성 |
| 경제/자동화형 | 오프라인 보상, 재화 드랍, 시작 자원 |

### 전투 반영 방식
- 메타에서 선택한 노드는 전투 시작 시 `SkillLoadoutState`, `AutoAttackProfile`, `EconomyModifier`, `SpecialRuleSet` 형태의 ECS 데이터로 주입된다.
- 즉, 메타 성장의 구현 목표는 UI 효과가 아니라 **전투 규칙 데이터 변경**이다.
- 전투 중 강화는 향후 구현 가능 구조로 열어 두되, 현재는 상세 수치와 발동 조건을 확정하지 않는다.

## 스킬 트리 카테고리
| 카테고리 | 정체성 | 예시 플레이 스타일 |
| --- | --- | --- |
| `Shift` | 이동 규칙과 레인 대응 확장 | 빠른 재배치형 |
| `Strike` | 자동공격 성능과 공격 범위 확장 | 공격 유지형 |
| `Flow` | 연속 처치와 콤보 흐름 강화 | 콤보 운영형 |
| `Automation` | 반복 플레이 편의와 부분 자동화 제공 | 방치 친화형 |
| `Economy` | 보상량, 오프라인 획득, 자원 변환 강화 | 누적 성장형 |

## 대표 노드 예시
| 카테고리 | 노드명 | 효과 | 설계 의도 |
| --- | --- | --- | --- |
| Shift | `Cross Lane Drift` | 이동 속도 증가 | 4레인 답답함 완화 |
| Shift | `Buffered Route` | 이동 예약 처리 안정성 보정 | 난상황 대응 |
| Strike | `Rapid Chamber` | 기본 공격 속도 증가 | 자동공격 유지력 강화 |
| Strike | `Pierce Seed` | 향후 관통 확장 가능 슬롯 해금 | 전투 확장 포인트 |
| Flow | `Streak Battery` | 콤보 유지 시 향후 전투 중 강화 연결 가능 | 콤보의 기능성 강화 |
| Flow | `Fallback Guard` | 라이프 손실 후 복구 여지 강화 | 실수 복구 |
| Automation | `Lane Assist` | 가장 가까운 위협 레인 보조 훅 제공 | 방치 감성 보조 |
| Automation | `Auto Salvage` | 미획득 보상 자동 회수 | 반복 피로 감소 |
| Economy | `Deep Cache` | 오프라인 보상 최대 시간 +2시간 | 방치형 루프 강화 |
| Economy | `Refine Cycle` | `Core Scrap` 일부를 `Pulse Data`로 전환 | 장기 성장 효율 |

## 초중후반 성장 구조

### 초반
| 목표 | 내용 |
| --- | --- |
| 조작 안정화 | 이동 속도, 입력 보정형 노드 우선 |
| 기본 루프 이해 | 첫 번째 트리 분기 2~3개만 제공 |
| 반복 동기 형성 | 1~2판 뒤 명확한 체감 변화 제공 |

### 중반
| 목표 | 내용 |
| --- | --- |
| 빌드 차별화 | `Shift`, `Strike`, `Flow` 중 2개 조합 강요 |
| 스테이지별 대응 | 특정 적 속도/체력 조합에 강한 노드 조합 등장 |
| 경제 선택 | 전투 효율 vs 오프라인 효율 갈등 유도 |

### 후반
| 목표 | 내용 |
| --- | --- |
| 특화 빌드 완성 | 고기동형, 공격 확장형, 안정적 자동화형 등 극단 빌드 허용 |
| 콘텐츠 해금 순환 | 보스 시질과 아카이브 키로 상위 패턴 개방 |
| 재도전 가치 유지 | 신규 규칙 노드와 시즌형 확장 여지 확보 |

## 프레스티지/환생 필요 여부

### 권장 결론
- **MVP에서는 환생을 넣지 않는다.**
- 대신 중후반용 “리셋 전제의 압축 성장” 구조는 확장 옵션으로만 남긴다.

### 이유
- 현재 핵심 가설은 자동공격 생존전 손맛과 반복 성장의 결합 검증이다.
- 환생 시스템은 경제 구조를 크게 흔들고, 전투 감각 검증을 흐릴 위험이 있다.
- 첫 버전은 스킬 트리 확장과 난도 해금만으로도 반복 동기를 충분히 확인할 수 있다.

### 후속 확장안
- 시즌 2 이후 `Recompile` 같은 재시작 메커니즘을 도입할 수 있다.
- 단, 환생 보상은 단순 배율보다 **새로운 규칙 슬롯 해금**에 연결하는 것이 낫다.

## 방치형 보상 적용 여부

### 적용 결론
- **적용한다. 단, 능동 플레이를 대체하지 않는 보조 구조로 제한한다.**

### 적용 규칙 예시
| 항목 | 규칙 |
| --- | --- |
| 최대 누적 시간 | 8시간 |
| 효율 | 능동 플레이 대비 25~35% |
| 산출 재화 | `Core Scrap`, 일부 `Idle Charge` 중심 |
| 제외 항목 | 희귀 보스 재화, 핵심 해금 조각 |
| 메타 연동 | `Economy`/`Automation` 트리로 효율 증대 |

### 설계 이유
- 방치형 감성은 유지하되, 핵심 재미인 레인 운영과 생존 긴장을 대체하면 안 된다.
- 따라서 오프라인 보상은 **전투 재진입을 빠르게 돕는 윤활제** 역할만 맡는다.

## 성장 밸런스 리스크
| 리스크 | 설명 | 대응 |
| --- | --- | --- |
| 수치 노드가 규칙 노드를 압도 | 결국 공격력만 올리는 게임이 될 수 있음 | 규칙 노드 선행 해금, 수치 노드 비중 제한 |
| 자동화 노드가 입력 재미를 제거 | 방치형이 전투를 먹어버릴 수 있음 | 자동화는 보정/보조 위주로 제한 |
| 오프라인 보상이 능동 플레이 가치 훼손 | 접속 동기가 약화될 수 있음 | 희귀 보상은 능동 플레이 전용으로 유지 |
| 트리 선택이 정답화 | 특정 빌드만 강하면 다양성이 사라짐 | 적 속도/체력 조합 다양화, 리스펙 비용 완화 |
| 중후반 성장 체감 저하 | 누적이 둔해지면 이탈 가능 | 큰 규칙 변조 노드를 구간별 배치 |
| 3D 스킨이 가독성을 해칠 수 있음 | 전투 테마가 바뀔수록 적/레인 구분이 약해질 수 있음 | 실루엣과 레인 발광 규칙을 공통 가이드로 유지 |

## 메타 데이터 구조 메모
| 데이터 | 저장 방식 제안 |
| --- | --- |
| 노드 정의 | ScriptableObject authoring -> BlobAsset 변환 |
| 플레이어 해금 상태 | 세이브 데이터(JSON/Binary) |
| 전투 반영값 | 전투 시작 시 ECS singleton 또는 초기 컴포넌트 주입 |
| 경제 곡선 | 버전 관리 가능한 데이터 테이블 + BlobAsset |
| 3D 전투 스킨 규칙 | 머티리얼/실루엣/레인 발광 가이드 데이터로 관리 가능 |

## 실행 우선순위
1. `Shift`, `Strike`, `Economy` 3개 카테고리만으로 첫 트리 버전을 만든다.
2. 첫 노드는 이동 속도, 공격 속도, 오프라인 효율처럼 체감 큰 규칙 변화형으로 배치한다.
3. 수치 증폭 노드는 서브 트리 후반부로 미룬다.
4. 환생은 MVP 범위에서 제외하고 오프라인 보상 상한만 먼저 검증한다.
