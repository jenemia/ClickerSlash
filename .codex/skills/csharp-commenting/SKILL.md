# C# 주석 정책

코드 주석을 추가하거나 수정할 때 `Assets/` 아래의 프로젝트 소유 `.cs` 파일에 이 정책을 적용합니다.

## 필수 주석 범위

- 각 클래스, 구조체, 의미 있는 중첩 타입에는 짧은 XML 요약 주석을 추가합니다.
- 각 메서드에는 게임플레이 흐름 또는 툴링 흐름에서 어떤 역할을 하는지 설명하는 짧은 XML 요약 주석을 추가합니다.
- 불리언 또는 불리언처럼 쓰는 플래그(`bool`, 불리언 의미의 `byte`)에는 `true/false` 또는 `0/1`이 도메인 관점에서 무엇을 뜻하는지 설명하는 주석을 추가합니다.
- 메서드 내부에서는 의도가 바로 읽히지 않는 분기나 상태 전이 앞에 짧은 인라인 주석을 추가해, 다음 작성자가 흐름을 다시 추론하지 않아도 이해할 수 있게 합니다.

## 스타일 규칙

- 문법을 그대로 풀어쓴 설명보다 의도를 설명하는 주석을 우선합니다.
- 주석은 짧고 구체적으로 유지합니다.
- 타입과 메서드에는 IDE 호버 정보가 살아 있도록 XML 문서 주석을 사용합니다.
- 로컬 로직, 가드 절, 플래그 의미에는 일반 `//` 주석을 사용합니다.
- 메서드가 여러 단계로 나뉘면 모든 줄에 주석을 다는 대신 단계 경계에 주석을 답니다.

## 우선 설명할 대상

- ECS 시스템: 업데이트 목적, 조기 종료 조건, 상태 전이가 왜 일어나는지 설명합니다.
- Authoring/Baker 클래스: 어떤 authoring 데이터를 노출하고 어떤 런타임 ECS 데이터를 만들어내는지 설명합니다.
- UI 프레젠터: UI가 언제 런타임 상태를 읽는지, 각 액션이 무엇을 트리거하는지 설명합니다.
- 테스트: 어떤 시나리오를 검증하는지, 각 준비 단계가 왜 필요한지 설명합니다.

## 피할 것

- 코드 이름을 문장으로 다시 풀어쓴 수준의 주석
- 단순 getter/setter에 붙는 형식적인 주석
- 1~2줄로 충분한데도 길게 늘어진 블록 주석
